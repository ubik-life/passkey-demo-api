# CLAUDE.md — Контекст проекта для агента

## Проект

**Passkey Demo API** — Go-сервер, реализующий полный цикл авторизации без паролей (WebAuthn + JWT + SQLite). Первый кирпич ноды Ubik.

UI-часть: `git@github.com:ubik-life/passkey-demo-ui.git`
Концепция платформы: `git@github.com:ubik-life/concept.git`

## Документирование процесса

Этот проект разрабатывается как обучающий материал. Каждая сессия работы с агентом фиксируется в `devlog/`:

```
devlog/
  00-setup.md        ← стартовая точка: как начать проект с нуля
  00-intent.md       ← зачем строим, API-контракт, решения по архитектуре
  01-api-contract.md ← OpenAPI-спека
  02-gherkin.md      ← компонентные тесты
  03-go-server.md    ← TDD-цикл: Go-сервер
```

### Формат каждого файла devlog

```markdown
# NN — Название шага

## Промпт
Дословно что написал автор агенту.

## Что сделал агент
Краткое описание результата.

## Решения
Что обсуждалось, какие альтернативы рассматривались, почему выбрано именно это.

## Результат
Ссылка на файлы или коммит.
```

### Зачем

Цель — чтобы студент смог повторить весь путь шаг за шагом, работая с ИИ-агентом так же, как это делал автор. Журнал версионируется вместе с кодом.

## Принятые решения

- REST API: двухфазные POST вместо PUT для завершения регистрации/входа (см. `devlog/00-intent.md`)
- Термины `attestation` и `assertion` в URI — взяты из WebAuthn напрямую
- Trunk Based Development: ветки живут 1–2 дня
- Коммиты по Conventional Commits
- Спека в `api-specification/openapi.yaml` (OpenAPI 3.1, versioning через `/v1/`)
